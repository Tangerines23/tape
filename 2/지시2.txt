인터랙티브 회로도 뷰어 개발 요청서
1. 프로젝트 개요
목표: 총 10페이지 분량의 회로도 이미지 파일을 웹 환경에서 직관적이고 전문적으로 보여줄 수 있는 '스토리-플로우 필름 스트립' 형태의 뷰어 개발.

핵심 컨셉: 사용자가 전체 회로도의 맥락을 파악하면서도, 현재 보고 있는 페이지에 집중할 수 있는 UI/UX를 제공하는 것을 목표로 합니다.

2. 주요 기능 요구사항 (Functional Requirements)
2.1. 기본 레이아웃
메인 뷰 (Main View): 화면의 대부분을 차지하는 영역. 선택된 회로도 이미지가 크게 표시되어야 합니다.

필름 스트립 (Film Strip): 화면 하단에 위치하는 가로 스크롤 형태의 네비게이션 바. 모든 회로도 페이지(10개)의 썸네일 이미지가 나열되어야 합니다.

2.2. 기능 명세
초기 로딩: 페이지가 처음 로드될 때, 첫 번째(1페이지) 회로도가 메인 뷰에 표시되고, 하단 필름 스트립의 첫 번째 썸네일이 '활성화(active)' 상태로 표시되어야 합니다.

메인 이미지 전환:

메인 뷰의 이미지는 항상 선명하게 보여야 하며, 영역에 맞춰 크기가 조절되어야 합니다. (가로/세로 비율 유지)

페이지 전환 시, 이미지가 딱딱하게 바뀌는 것이 아니라 부드러운 페이드 인/아웃(fade in/out) 효과가 적용되어야 합니다.

네비게이션 기능 (3가지 방식 필수)

(1) 썸네일 클릭: 하단 필름 스트립의 특정 썸네일을 클릭하면, 해당 페이지가 메인 뷰에 즉시 표시되어야 합니다.

(2) 좌/우 화살표 버튼: 메인 뷰 좌우에 마우스를 올리면 나타나는 화살표 버튼 < > 을 클릭하여 이전/다음 페이지로 이동할 수 있어야 합니다. (1페이지에서는 < 비활성화, 10페이지에서는 > 비활성화)

(3) 키보드 : 키보드의 좌(A) / 우(D) 방향키를 눌러도 이전/다음 페이지로 동일하게 이동해야 합니다.

'활성화' 상태 표시:

현재 메인 뷰에 보이는 페이지에 해당하는 하단 썸네일은 시각적으로 명확하게 구분되어야 합니다. (예: 컬러 테두리, 배경색 변경, 크기 확대 등)

페이지가 전환될 때마다 이 '활성화' 표시는 해당 썸네일로 부드럽게 이동해야 합니다.

반응형 디자인:

데스크탑, 태블릿, 모바일 등 다양한 화면 크기에서 레이아웃이 깨지지 않고 최적화된 형태로 보여야 합니다.

3. 디자인 및 UI/UX 요구사항
전체적인 톤앤매너: 깔끔하고 현대적이며 전문적인 느낌.

애니메이션: 모든 상호작용(페이지 전환, 썸네일 활성화 등)은 0.3초 내외의 부드러운 전환 효과를 포함해야 합니다.

스타일 가이드:

모서리는 약간 둥글게 처리 (rounded corners)

주요 UI 요소에는 그림자 효과(box-shadow)를 적용하여 입체감 부여

버튼 및 썸네일에 마우스를 올렸을 때(hover) 시각적인 피드백이 있어야 함.

4. 전달 필요 에셋 (Assets)
회로도 이미지 파일 (총 10개):

파일 형식: PNG

파일명 규칙: 1.png, 2.png, ... , 10.png

모든 이미지 파일은 동일한 가로/세로 비율을 가집니다.

궁금한 점이 있다면 언제든지 문의 바랍니다.
